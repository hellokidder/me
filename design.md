# A股超短线辅助系统 —— 项目设计文档

## 一、策略概述

本项目旨在构建一套 **A股超短线（T+1）辅助分析系统**，帮助交易者在每个交易日盘前完成信息收集与分析，输出当日大概率大幅上涨的 4~6 只个股推荐，供人工参考执行。

核心流程：

```
盘前信息收集 → 大盘分析 → 候选股筛选 → AI 综合分析 → 生成操作建议报告
```

---

## 二、整体架构

```
[定时任务 - 每日 8:30 触发]
        ↓
[数据收集层 - AkShare]
├── 大盘指数、期货升贴水
├── 昨日涨停板、连板数据
├── 龙虎榜席位
├── 北向资金净流入/流出
└── 财经新闻 RSS
        ↓
[初筛层 - 纯 Python 规则]
├── 量比、换手率过滤
├── 技术形态筛选
└── 输出候选股列表（10~15 只）
        ↓
[分析层 - Claude Sonnet 4.5]
├── 新闻热点识别
├── 题材匹配候选股
└── 输出带评分的中间结果
        ↓
[决策层 - Claude Opus 4.6]
├── 综合所有信息深度推理
├── 给出最终 4~6 只推荐
└── 每只附带推荐理由 + 风险提示
        ↓
[输出 - Markdown 报告 / 推送通知]
```

## 三、技术选型

| 层次 | 技术 | 选型理由 |
|------|------|----------|
| 编程语言 | Python | A股量化生态完善，AkShare/pandas/ta-lib 均为 Python 原生 |
| 行情数据 | AkShare（主）/ Tushare（备） | 免费、专为 A 股设计、社区活跃；备用源防止接口失效 |
| 数据处理 | pandas | 时间序列处理行业标准 |
| 技术指标 | pandas-ta | 开箱即用，覆盖主流技术指标 |
| AI 框架 | Anthropic Python SDK | 只用 Claude 系列模型，直接调用最简洁，无需中间层 |
| 主力模型 | Claude Sonnet 4.5 | 速度快、成本低，用于数据汇总与初步分析 |
| 决策模型 | Claude Opus 4.6 | 推理更深，用于最终综合判断与报告生成 |
| 数据存储 | SQLite | 轻量、零部署，适合本地单机运行的历史数据存储 |
| 配置管理 | .env + config.yaml | API Key 存 .env，业务参数存 config.yaml |

> **为什么不用 LangChain**：过度封装、API 变动频繁、调试困难。本项目核心流程是「获取数据 → 拼装 Prompt → 调用 LLM → 解析输出」，直接用 SDK 更干净可控。
>
> **为什么不用 LiteLLM**：项目只使用 Claude 模型，Anthropic 官方 SDK 更直接，减少一层抽象，调试更简单。

## 四、信息收集维度

### 4.1 宏观 / 情绪面

- 隔夜美股、港股表现
- 沪深 300 期货升贴水
- 北向资金昨日净流入/流出
- 融资融券余额变化
- 市场情绪指标：涨停板数量、炸板率

### 4.2 题材 / 热点面

- 昨日涨停股连板情况（龙头股是否延续）
- 当日早盘竞价异动股
- 财经新闻热点（政策、行业事件）
- 游资/机构席位动向（龙虎榜数据）

## 五、选股标准（量化维度）

| 维度 | 示例标准 |
|------|----------|
| 技术形态 | 缩量回调后放量突破、涨停板次日开板回踩 |
| 量能 | 昨日成交量 ≥ 5 日均量的 2 倍 |
| 量比 | 开盘量比 ≥ 2.0 |
| 换手率 | 3%~15%（过低无人气，过高有出货嫌疑） |
| 位置 | 区分低位首板（低风险）vs 高位接力（高风险） |
| 题材强度 | 是否有持续性政策催化 |
| 竞价表现 | 9:15~9:25 竞价量价配合（见 5.1 二次筛选流程） |

### 5.1 两阶段筛选流程

由于集合竞价数据在 9:15 才产生，选股分为两个阶段：

| 阶段 | 触发时间 | 数据来源 | 输出 |
|------|----------|----------|------|
| 盘前静态分析 | 8:30 | 昨日行情、涨停板、龙虎榜、新闻 | 候选股池（10~15 只） |
| 竞价二次筛选 | 9:25 | 集合竞价量价数据 | 最终推荐（4~6 只） |

> 竞价阶段为可选增强模块，Phase 1~3 先跑通盘前静态分析流程。

## 六、买卖执行规范

### 6.1 买入

- 明确是集合竞价买入还是开盘后追涨（超短线差别显著）
- 设置涨幅阈值：开盘涨超 5% 则放弃追入（避免高位接盘）
- 仓位分配：4~6 只个股均衡分仓

### 6.2 卖出

- 卖出时机：开盘集合竞价卖 or 开盘后高抛，需提前设定规则
- 止损规则：当日跌超 3% 强制止损
- 可保留部分仓位的半仓策略

---

## 七、风控体系

| 风控项 | 规则 |
|--------|------|
| 单日最大亏损 | 总仓位亏损超过 5% 停止当日操作 |
| 连亏熔断 | 连续亏损 3 天后暂停策略，进行复盘 |
| 黑天鹅处理 | 突发退市风险、ST 预警时强制出清 |
| 流动性风险 | 小票仓位上限控制，避免买入影响价格 |

## 八、模型分工详细说明

### Claude Sonnet 4.5（分析层）

- 输入：结构化候选股数据 + 今日财经新闻
- 任务：识别热点题材、匹配候选股、输出带评分的中间结果
- 选用理由：速度快、成本低，处理大量信息汇总够用

### Claude Opus 4.6（决策层）

- 输入：Sonnet 的中间结果 + 完整上下文
- 任务：深度推理，给出最终 4~6 只推荐，附推荐理由和风险提示
- 选用理由：推理能力更强，适合最关键的决策环节，长上下文 200K

> 两者均通过 **LiteLLM** 桥接调用，切换模型成本极低。

## 九、输出报告结构

每日生成一份 Markdown 格式的操作建议报告，结构如下：

```markdown
# {日期} 今日操作建议

## 大盘情绪评分：X / 10
（评分说明：综合期货升贴水、北向资金、昨日涨停情况等）

## 今日热点题材
- 题材一：xxx（政策催化/事件驱动）
- 题材二：xxx

## 推荐个股

### 1. 股票名称（代码）
- 推荐理由：xxx
- 关注价位：xxx
- 风险提示：xxx

...

## 风险提示
今日大盘情绪 X 分，建议仓位控制在 X% 以内。
```

## 十一、数据存储方案

### 11.1 SQLite 表设计

| 表名 | 用途 |
|------|------|
| daily_market | 每日大盘数据（指数、北向资金、情绪指标） |
| candidates | 每日候选股及筛选得分 |
| recommendations | 每日最终推荐股及 AI 评分 |
| trade_records | 实际交易记录（人工录入，用于复盘） |
| performance | 推荐 vs 实际表现对比 |

### 11.2 报告归档

每日报告以 `output/YYYY-MM-DD.md` 格式保存，便于回溯和复盘。

## 十三、回测与验证机制

### 13.1 验证指标

| 指标 | 定义 |
|------|------|
| 推荐命中率 | 推荐股次日收盘涨幅 > 0% 的比例 |
| 平均收益率 | 所有推荐股次日涨幅的算术平均 |
| 最大单日回撤 | 单日推荐组合的最大亏损幅度 |
| 超额收益 | 推荐组合收益 vs 同期沪深 300 涨幅 |

### 13.2 每日记录流程

1. 盘前生成推荐报告
2. 次日收盘后自动拉取推荐股的实际表现
3. 写入 `performance` 表，计算当日各项指标
4. 每周生成一份周度复盘摘要

> Phase 4 的人工验证期（2~4 周）内，不投入真金白银，仅做模拟记录。

## 十四、异常处理与日志

### 14.1 数据层容错

| 异常场景 | 处理策略 |
|----------|----------|
| AkShare 接口失败 | 自动切换 Tushare 备用源，最多重试 3 次 |
| 数据缺失（非交易日等） | 检测并跳过，记录日志 |
| 数据异常值 | 基本校验（如涨跌幅超 ±20% 标记异常） |

### 14.2 AI 层容错

| 异常场景 | 处理策略 |
|----------|----------|
| Opus 调用失败/超时 | 降级为 Sonnet 生成最终报告，标记为「降级模式」 |
| 输出格式异常 | 重试一次，仍失败则输出原始文本供人工阅读 |
| API 额度耗尽 | 告警通知，跳过 AI 分析，仅输出规则筛选结果 |

### 14.3 日志规范

- 使用 Python `logging` 模块，日志写入 `logs/YYYY-MM-DD.log`
- 级别：INFO（正常流程）、WARNING（降级/重试）、ERROR（失败）
- 关键节点记录耗时，便于性能分析

## 十五、开发路线图

| 阶段 | 内容 | 目标 |
|------|------|------|
| Phase 1 | 数据收集层（AkShare） | 跑通数据抓取，验证数据完整性 |

## 十六、注意事项

1. **本系统仅作辅助分析，最终决策由人工执行**，不做自动下单
2. AI 存在幻觉风险，需人工核实关键信息（如重大利好是否真实）
3. 超短线策略胜率受市场风格影响大，需持续复盘迭代
4. 建议保留每日交易记录，用于后续策略优化
